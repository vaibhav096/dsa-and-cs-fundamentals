Problem 1: Partition Equal Subset Sum
Difficulty: Medium
Topic: Dynamic Programming (0/1 Knapsack Pattern)

-----------------------------------------------------------------------
PROBLEM STATEMENT:
Given an array `arr[]` of size `N` containing non-negative integers, determine if the array can be partitioned into two subsets such that the sum of elements in both subsets is equal.

Example 1:
Input: N = 4, arr = [1, 5, 11, 5]
Output: True
Explanation: The two subsets are [1, 5, 5] and [11]. Both sum to 11.

Example 2:
Input: N = 3, arr = [1, 3, 5]
Output: False
Explanation: Total sum is 9. We cannot split it into two subsets of sum 4.5.

-----------------------------------------------------------------------
THINKING PROCESS & APPROACH:

1.  **Fundamental Check**: For two subsets to have equal sum, the TOTAL sum of the array must be even. If `sum(arr)` is odd, we can immediately return `False` because you can't divide an odd number into two equal integers.

2.  **Target Identification**: If the total sum is `S`, we are essentially looking for ONE subset that sums up to `S / 2`. If we find one, the remaining elements naturally form the other subset with the same sum.
    Target `K = S / 2`.

3.  **Recursion -> DP Check**:
    For each element, we have two choices:
    -   **Pick it**: Subtract its value from the target.
    -   **Leave it**: Keep the target same.
    This structure is identical to the **0/1 Knapsack Problem**. We want to fill a "knapsack" of capacity `K` using correct items.

4.  **DP Table Definition**:
    Let `dp[i][j]` be a boolean value indicating if a sum of `j` is possible using the first `i` items.
    
    State Transition:
    `dp[i][j] = dp[i-1][j]` (Don't include item `i`) 
              `OR dp[i-1][j - arr[i]]` (Include item `i`, if `j >= arr[i]`)

5.  **Space Optimization**:
    Notice `dp[i]` only depends on `dp[i-1]`. We can reduce this to a 1D array `dp[j]`.
    `dp[j]` = True if sum `j` is possible.
    **Important**: When iterating with a 1D array for 0/1 Knapsack, we must iterate BACKWARDS from `K` to `arr[i]` to avoid using the same item multiple times for the same sum in the same step.

    Algorithm:
    1. Calculate `total_sum`. If odd -> False.
    2. `target = total_sum // 2`
    3. Initialize `dp` array of size `target + 1` with False.
    4. `dp[0] = True` (Sum 0 is always possible with empty set).
    5. For each number `num` in `arr`:
         For `j` from `target` down to `num`:
             `dp[j] = dp[j] or dp[j - num]`
    6. Return `dp[target]`.

-----------------------------------------------------------------------
COMPLEXITY:
-   Time: O(N * Target), where Target is TotalSum/2.
-   Space: O(Target) for the optimized 1D DP array.
