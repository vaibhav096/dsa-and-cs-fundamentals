================================================================================
                    DISK SCHEDULING
================================================================================

Disk scheduling algorithms determine the order to service disk I/O requests.


================================================================================
                    1. DISK STRUCTURE
================================================================================

HARD DISK COMPONENTS:

         Spindle
            │
    ┌───────┼───────┐
    │   ┌───┴───┐   │
    │   │Platter│   │ ← Spinning disk
    │   │       │   │
    │   └───────┘   │
    │    ↑          │
    │  Read/Write   │
    │    Head       │
    └───────────────┘
           ↑
       Arm (seeks)


TERMS:

- Platter: Circular disk coated with magnetic material
- Track: Concentric circle on platter
- Sector: Pie-slice portion of track (typically 512B or 4KB)
- Cylinder: Same track on all platters
- Head: Reads/writes data
- Arm: Moves head across platters


DISK ADDRESS:
(Cylinder, Head, Sector) or just block number


================================================================================
                    2. DISK ACCESS TIME
================================================================================

Total Time = Seek Time + Rotational Latency + Transfer Time


1. SEEK TIME:
   - Time to move head to correct cylinder
   - SLOWEST component
   - Typically 3-15 ms

2. ROTATIONAL LATENCY:
   - Time for sector to rotate under head
   - Average = 1/2 rotation time
   - 7200 RPM → 4.17 ms average

3. TRANSFER TIME:
   - Time to read/write data
   - Depends on rotation speed and data size


GOAL:
Minimize seek time (most significant component).


================================================================================
                    3. DISK SCHEDULING ALGORITHMS
================================================================================

Example Queue: 98, 183, 37, 122, 14, 124, 65, 67
Head starts at: 53
Disk range: 0-199


--------------------------------------------------------------------------------
3.1 FCFS (First Come First Served)
--------------------------------------------------------------------------------

Service requests in order of arrival.

Order: 53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67

Head Movement:
|45| + |85| + |146| + |85| + |108| + |110| + |59| + |2| = 640 cylinders

    0                    53        98                     183   199
    |--------------------*---------*-----------------------*-----|
                         Start

Simple but not optimal.


--------------------------------------------------------------------------------
3.2 SSTF (Shortest Seek Time First)
--------------------------------------------------------------------------------

Service nearest request first.

From 53: Nearest is 65
From 65: Nearest is 67
From 67: Nearest is 37
... and so on

Order: 53 → 65 → 67 → 37 → 14 → 98 → 122 → 124 → 183

Head Movement:
|12| + |2| + |30| + |23| + |84| + |24| + |2| + |59| = 236 cylinders

Better than FCFS, but can cause STARVATION.


--------------------------------------------------------------------------------
3.3 SCAN (Elevator Algorithm)
--------------------------------------------------------------------------------

Head moves in one direction, servicing all requests.
Reverses at end, continues servicing.

Like elevator: goes up, then down.

Direction: Toward 0

Order: 53 → 37 → 14 → 0 → 65 → 67 → 98 → 122 → 124 → 183

    0        14  37    53  65 67     98    122 124           183
    *--------*---*------*---*-*------*------*---*-------------*
    ←←←←←←←←←←←←←←←←←← then →→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→

Head Movement:
|16| + |23| + |14| + |65| + |2| + |31| + |24| + |2| + |59| = 236 cylinders

No starvation, uniform wait time.


--------------------------------------------------------------------------------
3.4 C-SCAN (Circular SCAN)
--------------------------------------------------------------------------------

Head moves in one direction only.
At end, jumps to beginning without servicing.

Like circular elevator: always goes up, takes stairs down.

Direction: Toward 199

Order: 53 → 65 → 67 → 98 → 122 → 124 → 183 → 199 → 0 → 14 → 37

    0    14  37    53  65 67     98    122 124           183  199
    *----*---*------*---*-*------*------*---*-------------*----*
                    →→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→
    ←←←←←←←←←←←←←←←←  (jump, no service)

Head Movement (excluding jump):
53→65→67→98→122→124→183→199 = 146
0→14→37 = 37
Total = 183 cylinders (+ jump)

More uniform wait time than SCAN.


--------------------------------------------------------------------------------
3.5 LOOK
--------------------------------------------------------------------------------

Like SCAN but doesn't go to end.
Reverses at last request in that direction.

Direction: Toward 0

Order: 53 → 37 → 14 → 65 → 67 → 98 → 122 → 124 → 183

    0    14  37    53  65 67     98    122 124           183
         *---*------*---*-*------*------*---*-------------*
         ←←←←←←←←←←← then →→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→→

Head Movement:
|16| + |23| + |51| + |2| + |31| + |24| + |2| + |59| = 208 cylinders


--------------------------------------------------------------------------------
3.6 C-LOOK (Circular LOOK)
--------------------------------------------------------------------------------

Like C-SCAN but doesn't go to end.
Jumps from last request to first request.

Direction: Toward 199

Order: 53 → 65 → 67 → 98 → 122 → 124 → 183 → 14 → 37

Head Movement:
53→183 = 130
14→37 = 23
Total = 153 cylinders (+ jump from 183 to 14)


================================================================================
                    4. ALGORITHM COMPARISON
================================================================================

+------------+-----------------+------------+------------+
| Algorithm  | Seek Time       | Starvation | Uniformity |
+------------+-----------------+------------+------------+
| FCFS       | High (640)      | No         | Fair       |
| SSTF       | Low (236)       | Yes        | Unfair     |
| SCAN       | Medium (236)    | No         | Good       |
| C-SCAN     | Medium (183)    | No         | Best       |
| LOOK       | Low (208)       | No         | Good       |
| C-LOOK     | Lowest (153)    | No         | Best       |
+------------+-----------------+------------+------------+


SELECTION FACTORS:
- Request distribution
- I/O load
- Seek time vs latency priority


================================================================================
                    5. DISK MANAGEMENT
================================================================================

1. DISK FORMATTING:

   Low-Level (Physical):
   - Divide disk into sectors
   - Fill with data pattern
   - Done at factory

   Logical (High-Level):
   - Create file system
   - Initialize directory structure
   - Create free space list


2. BOOT BLOCK:

   - First sector(s) of disk
   - Contains boot code
   - MBR (Master Boot Record) or GPT


3. BAD BLOCK MANAGEMENT:

   - Disk has defective sectors
   - Handled by controller or OS
   - Sector sparing: Redirect to spare
   - Sector slipping: Shift sectors


================================================================================
                    6. SWAP SPACE MANAGEMENT
================================================================================

Swap space: Disk area used as virtual memory extension.

LOCATION OPTIONS:

1. File System File:
   - Swap file in regular file system
   - Flexible but slower
   - Windows pagefile.sys

2. Separate Partition:
   - Dedicated partition
   - Faster (no file system overhead)
   - Linux swap partition


MANAGEMENT:
- Track which pages swapped out
- Handle swap in/out efficiently


================================================================================
                    7. RAID (Redundant Array of Independent Disks)
================================================================================

Multiple disks working together for performance and/or reliability.


RAID 0 (Striping):
- Data split across disks
- No redundancy
- Best performance
- One disk fails = all data lost

    Disk 1: A1, A3, A5...
    Disk 2: A2, A4, A6...


RAID 1 (Mirroring):
- Data duplicated on two disks
- Read from either
- Write to both
- 50% storage efficiency

    Disk 1: A1, A2, A3...
    Disk 2: A1, A2, A3... (copy)


RAID 5 (Striping with Parity):
- Data and parity distributed across disks
- Can survive one disk failure
- Good balance of performance and reliability

    Disk 1: A1, B1, C1, Dp
    Disk 2: A2, B2, Cp, D1
    Disk 3: A3, Bp, C2, D2
    Disk 4: Ap, B3, C3, D3

    p = parity (XOR of others)


RAID 6:
- Two parity blocks
- Can survive two disk failures


RAID 10 (1+0):
- Stripe of mirrors
- High performance + redundancy
- Needs minimum 4 disks


================================================================================
                    8. SSD vs HDD
================================================================================

+------------------+------------------+------------------+
| Feature          | HDD              | SSD              |
+------------------+------------------+------------------+
| Technology       | Magnetic disk    | Flash memory     |
| Moving parts     | Yes              | No               |
| Seek time        | 3-15 ms          | ~0.1 ms          |
| Sequential read  | 100-200 MB/s     | 500+ MB/s        |
| Random access    | Slow             | Fast             |
| Durability       | Fragile          | Robust           |
| Power            | Higher           | Lower            |
| Cost per GB      | Lower            | Higher           |
| Lifespan         | Longer (MTBF)    | Limited writes   |
+------------------+------------------+------------------+

SSD SCHEDULING:
- No seek time, so FCFS often used
- Wear leveling more important
- TRIM command for garbage collection


================================================================================
                    9. INTERVIEW QUESTIONS
================================================================================

Q1: What is disk scheduling?
A: Determining order to service disk I/O requests.
   Goal: Minimize seek time.

Q2: Explain SSTF algorithm.
A: Shortest Seek Time First. Service nearest request.
   Efficient but can cause starvation.

Q3: What is SCAN algorithm?
A: Elevator algorithm. Head moves in one direction,
   services all requests, reverses at end.

Q4: Difference between SCAN and C-SCAN?
A: SCAN reverses at end.
   C-SCAN jumps to start, only services in one direction.
   C-SCAN has more uniform wait times.

Q5: What is RAID?
A: Redundant Array of Independent Disks.
   Multiple disks for performance/reliability.
   RAID 0 (stripe), RAID 1 (mirror), RAID 5 (parity).

Q6: Why doesn't SSD need disk scheduling like HDD?
A: SSD has no mechanical parts.
   No seek time, random access as fast as sequential.
